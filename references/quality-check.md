# Quality Check — Validação de HTML Premium

Use este checklist para validar cada HTML antes de considerar finalizado.
Todo item deve ser aprovado. Se falhar em qualquer critério, refine antes de entregar.

---

## 1. Tese e Narrativa

- [ ] A tese central é clara e aparece nos primeiros 10 segundos
- [ ] O headline é forte, memorável e entrega uma promessa intelectual
- [ ] Existe um arco narrativo com abertura, desenvolvimento, clímax e fechamento
- [ ] Cada seção tem um propósito claro no arco narrativo
- [ ] As transições entre seções são fluidas
- [ ] O material defende um ponto de vista (não é neutro genérico)
- [ ] O fechamento provoca ação ou reflexão

## 2. Conteúdo e Escrita

- [ ] Português natural e fluido — não parece traduzido ou robótico
- [ ] Linguagem executiva sem ser fria ou distante
- [ ] Zero clichês ("no mundo atual", "transformação digital", "novo normal")
- [ ] Zero frases vazias ("é fundamental", "nesse contexto", "vale ressaltar")
- [ ] Cada parágrafo carrega informação ou argumento — nenhum é preenchimento
- [ ] Dados e exemplos são específicos, não genéricos
- [ ] O material ensina algo que o leitor não sabia ou não pensava assim
- [ ] O tom é consistente do início ao fim

## 3. Design Visual

- [ ] Hierarquia visual clara (título > subtítulo > corpo > detalhe)
- [ ] Paleta intencional — máximo 3-4 cores com propósito definido
- [ ] Contraste suficiente para leitura confortável (WCAG AA mínimo)
- [ ] Tipografia legível em todos os tamanhos de tela
- [ ] Espaçamento generoso — respiro visual entre elementos
- [ ] Alinhamento consistente e grid limpo
- [ ] Nenhum elemento puramente decorativo sem função
- [ ] Estética premium — não parece template gratuito
- [ ] Atmosfera coerente com o tema
- [ ] Hero section com impacto visual forte

## 4. Interatividade

- [ ] Cada elemento interativo ensina ou revela algo
- [ ] Nenhuma interação é puramente decorativa
- [ ] Feedback visual claro em hover, click e estados ativos
- [ ] Transições suaves (200-400ms)
- [ ] Scroll behavior coerente e confortável
- [ ] Sem scroll hijacking agressivo
- [ ] Sem carrosséis automáticos
- [ ] Interações funcionam em mobile (touch-friendly)
- [ ] Interações complementam a narrativa (não competem com ela)

## 5. Técnico

- [ ] HTML5 semântico (section, article, header, main, footer)
- [ ] Meta tags completas (charset, viewport, title, description)
- [ ] Responsivo — funciona em mobile, tablet e desktop
- [ ] Animações respeitam `prefers-reduced-motion`
- [ ] JavaScript sem erros no console
- [ ] CSS sem overflows ou quebras de layout
- [ ] Fontes carregam corretamente (Google Fonts com fallback)
- [ ] Performance aceitável — sem jank ou lag em scroll
- [ ] Código limpo e organizado
- [ ] Acessibilidade básica (alt texts, aria-labels, navegação por teclado)

## 6. Experiência Completa

- [ ] O material se sustenta como experiência independente
- [ ] Cria sensação de descoberta progressiva
- [ ] O leitor sai com um framework ou modelo mental novo
- [ ] O visual reforça o conteúdo (não apenas decora)
- [ ] A interatividade reforça a narrativa (não apenas anima)
- [ ] O material merece ser salvo, revisitado e compartilhado
- [ ] Funciona como referência — não como conteúdo descartável

## 7. Teste Final (o mais importante)

Responda honestamente:

- [ ] **Teste do screenshot**: se alguém vir só um screenshot, vai querer ver mais?
- [ ] **Teste da referência**: você salvaria este material nos seus bookmarks?
- [ ] **Teste do orgulho**: você apresentaria isto em um palco?
- [ ] **Teste do diferencial**: este material se destaca de 95% do que existe online?
- [ ] **Teste da descoberta**: o leitor aprende algo que não sabia ou não pensava assim?

---

## Classificação

Após a validação, classifique:

| Nível | Critério |
|-------|----------|
| **A+** | Todos os itens aprovados, teste final 5/5 |
| **A** | Todos os itens obrigatórios, teste final 4/5 |
| **B** | Pequenos ajustes necessários em 1-2 categorias |
| **C** | Refazer — múltiplas categorias com falhas |

**Só entregue materiais classificados como A ou A+.**
