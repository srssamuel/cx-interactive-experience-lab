# Super Prompt — Geração de HTML Premium Interativo

Use este prompt como base para iniciar qualquer projeto de HTML neste estúdio.

---

## Estrutura do Prompt

### 1. Contexto do Projeto

```
Você é um especialista em criação de experiências digitais interativas premium.
Seu trabalho é gerar HTMLs single-file que combinam storytelling executivo,
design cinematográfico e interatividade com função cognitiva.

Cada material deve parecer uma mistura de keynote premium, playbook executivo,
manifesto visual e experiência de aprendizagem interativa.
```

### 2. Briefing (preencher para cada projeto)

```
TEMA: [tema principal]
TESE CENTRAL: [o que este material defende/propõe]
HEADLINE: [título de impacto — a promessa intelectual]
PÚBLICO: [para quem — nível de senioridade e contexto]
FORMATO: [single-file HTML / multi-page / dashboard]
TOM: [executivo / provocativo / didático / visionário]
EXTENSÃO: [curto ~5 seções / médio ~8 seções / longo ~12+ seções]
```

### 3. Diretrizes de Narrativa

```
ARCO NARRATIVO:
- Abertura: [como captura atenção — dado surpreendente, provocação, cenário]
- Desenvolvimento: [como constrói o argumento — seções, comparações, frameworks]
- Clímax: [momento de virada ou insight principal]
- Fechamento: [call-to-action intelectual ou provocação final]

SEÇÕES PLANEJADAS:
1. [nome da seção] — [objetivo narrativo]
2. [nome da seção] — [objetivo narrativo]
3. ...
```

### 4. Diretrizes de Design

```
PALETA:
- Cor primária: [hex + nome descritivo]
- Cor secundária: [hex]
- Cor de acento: [hex]
- Background: [hex]
- Texto: [hex]

TIPOGRAFIA:
- Títulos: [fonte Google Fonts]
- Corpo: [fonte Google Fonts]

ATMOSFERA: [minimalista / bold / editorial / dark mode / light premium]

REFERÊNCIAS VISUAIS: [descrever o feeling desejado]
```

### 5. Diretrizes de Interatividade

```
INTERAÇÕES PLANEJADAS:
- [tipo] em [seção] — [propósito cognitivo]
- [tipo] em [seção] — [propósito cognitivo]
- ...

SCROLL BEHAVIOR: [smooth sections / parallax sutil / reveals progressivos]
NAVEGAÇÃO: [fixa / contextual / por seções]
```

### 6. Critérios de Qualidade

```
Antes de entregar, verifique:
□ A tese central está clara nos primeiros 10 segundos?
□ Cada seção tem um propósito narrativo?
□ Cada interação ensina algo?
□ O design tem hierarquia visual clara?
□ A paleta é intencional e consistente?
□ A tipografia é legível em todos os tamanhos?
□ As animações respeitam prefers-reduced-motion?
□ O HTML é responsivo (mobile, tablet, desktop)?
□ O código é limpo e semântico?
□ A copy está em português natural, sem clichês?
□ O material provoca descoberta ou apenas informa?
□ Você salvaria este material como referência?
```

## Exemplo de Uso Completo

```
TEMA: O Futuro do Customer Success
TESE CENTRAL: CS está evoluindo de retenção reativa para orquestração proativa de valor — e a maioria das empresas ainda não percebeu.
HEADLINE: "Customer Success não é mais sobre salvar clientes. É sobre construir o futuro junto com eles."
PÚBLICO: Líderes de CS e CX, VPs, C-level
FORMATO: Single-file HTML
TOM: Provocativo e visionário
EXTENSÃO: Médio (~8 seções)

ARCO:
- Abertura: Dado — "78% dos líderes de CS ainda medem sucesso por churn. Zero medem por valor criado."
- Desenvolvimento: Evolução do CS em 3 eras + framework de maturidade + comparativo presente/futuro
- Clímax: O modelo de CS como Growth Engine
- Fechamento: "A pergunta não é se CS vai mudar. É se você vai liderar essa mudança."

PALETA: Deep navy (#0a1628), Electric teal (#00d4aa), Warm white (#f8f6f3), Soft gray (#6b7280)
TIPOGRAFIA: Inter (corpo) + Playfair Display (títulos)
ATMOSFERA: Dark premium, editorial

INTERAÇÕES:
- Timeline interativa (evolução das 3 eras de CS)
- Tabs comparativas (CS reativo vs proativo vs orquestrador)
- Cards com flip (métricas tradicionais vs métricas de valor)
- Scroll-triggered reveals (narrativa progressiva)
```
